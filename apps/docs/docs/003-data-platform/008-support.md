# Support & Issue Triage

openJII is an open-source, non-profit platform built by the [Jan IngenHousz Institute](https://www.jan-ingenhousz-institute.org/). We are committed to supporting our users and community, but we are a small team — please read below to understand how support works.

## How to report issues

The fastest way to get help is to open a GitHub issue:

- **Bug reports:** [open a bug report](https://github.com/Jan-IngenHousz-Institute/open-jii/issues/new?template=bug_report.md)
- **Feature requests:** [post an idea](https://github.com/Jan-IngenHousz-Institute/open-jii/discussions/categories/ideas)
- **Security vulnerabilities:** do **not** open a public issue — follow our [security policy](https://github.com/Jan-IngenHousz-Institute/open-jii/security)

## How we triage

We use [Dosu](https://dosu.dev/) for automated issue triage. When you open a GitHub issue, Dosu helps categorise and prioritise it so our team can respond more effectively.

Our project management runs on [Linear](https://linear.app/openjii). GitHub issues are **synced bidirectionally with Linear** — any issue you create on GitHub automatically appears in our Linear backlog, and status updates flow back to GitHub. You don't need a Linear account; GitHub is the public-facing entry point.

## Response times

We do not offer an SLA or guaranteed response times. As an open-source non-profit project, our capacity varies. That said, we are doing our best to respond to issues promptly and support the researchers and developers who use our platform.

What you can generally expect:
- **Security issues** are treated with the highest priority.
- **Bugs that block core workflows** (measurements, data access, authentication) are prioritised next.
- **Feature requests and enhancements** are reviewed and planned into upcoming milestones as capacity allows.

## Contributing

If you'd like to help, contributions are welcome — whether that's code, documentation, bug reports, or ideas. See our [contributing guidelines](../licensing-contribution/contributing-guidelines) for how to get started.
