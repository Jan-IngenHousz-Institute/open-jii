## Use release notes in a study

When a platform change could affect collection or analysis:

1. record the mobile or web version used during the study;
2. note the measurement dates and relevant release date;
3. test a representative workflow before rolling an update to all field phones;
4. preserve protocol, workbook, and analysis versions with the exported data;
5. report any change in values or behavior with a small reproducible example.

GitHub's repository releases are useful for engineering artifacts and tags, but
the public web releases page is the researcher-facing source of truth.
