## Report a bug

1. Search the [openJII issue list](https://github.com/Jan-IngenHousz-Institute/open-jii/issues)
   for the same problem.
2. If it is new, [open a bug report](https://github.com/Jan-IngenHousz-Institute/open-jii/issues/new?template=bug_report.md).
3. Include the affected surface (web, mobile, data, or device), steps to
   reproduce, expected and actual behavior, time and timezone, app/browser
   version, and safe screenshots or logs.
4. For a data issue, include experiment and table identifiers plus a small
   non-sensitive example. Do not attach credentials or an unrestricted dataset.

## Suggest an improvement

Post a proposal in [GitHub Discussions: Ideas](https://github.com/Jan-IngenHousz-Institute/open-jii/discussions/categories/ideas).
Explain the researcher task, current workaround, who is affected, and what a good
outcome would look like. A concrete workflow is more useful than a feature name.

## Report a security vulnerability privately

Do **not** open a public issue. Follow the repository's
[Security policy](https://github.com/Jan-IngenHousz-Institute/open-jii/security)
and use **Report a vulnerability** from the Security tab.

<Callout type="warn">
  Never post passwords, six-digit sign-in codes, API keys, IoT certificates, private keys, session
  cookies, personal data, embargoed data, or precise sensitive locations in a public issue.
</Callout>

openJII is an open-source nonprofit project and does not publish a support SLA.
If a bug blocks authentication, measurement, sync, or data access, say so clearly
in the report so its impact is visible.
