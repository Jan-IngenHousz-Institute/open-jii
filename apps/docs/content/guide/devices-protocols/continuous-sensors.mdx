The canonical openJII product label is **Ambyte**. In source and API values it is
spelled `ambyte`. Older planning material also says “Ambit”; the current
interface, ingestion code, and sensor-family enum do not use that as the product
label.

Ambyte represents autonomous sensor hardware that records observations over
time. This differs from a handheld MultispeQ, where a person starts each reading.

## What you can do now

- Register Ambyte as an IoT device from **Devices** using its physical identifier.
- Issue device credentials from the registered device's **Credentials** tab.
- Upload an Ambyte folder to an experiment data table. The uploader expects an
  `Ambyte_N/[1-4]/*.txt` folder layout and processes the data asynchronously.
- View the resulting table after the processing pipeline has materialized it.

## Current limits

Ambyte is an ingestion family, but the protocol editor marks it **Coming Soon**
for local protocol execution. The registered-device **Monitoring** and
**Onboarding** tabs are also marked **Coming Soon**. Do not plan a field workflow
around a live monitoring dashboard until that workflow has been confirmed on the
deployed platform.
