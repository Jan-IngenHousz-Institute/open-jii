In openJII, the hardware collects a reading and a **protocol** tells that hardware
how to collect it. Choose both before field work so everyone in your experiment
uses the same method.

## Choose your route

- Use a [MultispeQ](/guide/devices-protocols/multispeq) for handheld,
  protocol-guided measurements taken by a person.
- Read [Continuous sensors](/guide/devices-protocols/continuous-sensors) if you
  are bringing Ambyte data into openJII or connecting an Ambit or MiniPAR to the
  web workbook host. Ambyte is an ingestion family; Ambit and MiniPAR are
  distinct locally identifiable families.
- Use [Protocols](/guide/devices-protocols/protocols) to choose an existing
  measurement recipe, create one, or fork someone else's recipe before editing.
- Use [Device commands](/guide/devices-protocols/commands) to send a single raw
  instruction (such as `hello` or `battery`) straight to a connected instrument.
- Use [Multi-device measurements](/guide/devices-protocols/multi-device) when
  several connected devices should run one shared cell or be routed to
  family-specific protocol or command cells.

<Callout type="info">
  Beyond the named families, openJII can technically talk to any instrument that speaks a line-based
  serial protocol, whether over USB serial or Bluetooth Low Energy (BLE). The **generic** device
  family covers both transports; the named families add device-specific identification and drivers
  on top.
</Callout>

<Callout type="info">
  openJII uses **device** for handheld hardware that a person operates and **sensor** for autonomous
  hardware that sends observations over time. The interface also uses **sensor family** as the
  shared technical category for MultispeQ, Ambyte, Ambit, MiniPAR, and generic hardware.
</Callout>

## Prepare before leaving connectivity

1. Attach a [workbook](/guide/experiments/workbooks) to the experiment.
2. Confirm that its protocol cells point to the intended protocols.
3. Open the experiment in the mobile app while online so its pinned workbook
   version is cached.
4. Connect the hardware and run a test measurement.

Then continue with [Taking measurements in the field](/guide/measuring/taking-measurements).
