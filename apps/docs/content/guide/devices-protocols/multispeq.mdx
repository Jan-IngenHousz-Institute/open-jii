A **MultispeQ** is handheld measurement hardware operated by a person. openJII
sends a selected MultispeQ protocol to the device, receives the raw response,
and can run the workbook's analysis macro before you accept the result.

Use it when your method calls for measurements at chosen plants, leaves, or time
points rather than an unattended data stream.

## Connect a device

Supported devices appear together in the **Devices** sheet, but Bluetooth and
USB setup follow different paths. MultispeQ is the most common case.

### Bluetooth

1. Turn on the device. To make a MultispeQ discoverable, hold its button for
   about three seconds.
2. In the openJII mobile app, open **Devices** from the device chip.
3. Allow Bluetooth access when Android asks.
4. Select **Scan**, then **Connect** beside the intended device.
5. Check that the sheet shows the device as connected.

### USB cable

1. Connect the device to the Android phone with a supported USB cable or
   adapter. Use a powered USB hub when connecting several devices.
2. Open **Devices** from the device chip and select **Connect** beside the wired
   device.
3. Approve Android's USB permission request when it appears.
4. Check that the sheet shows the device as connected.

The primary connected device can also show battery level and firmware version
when those values are available.

The mobile app connects over Bluetooth and wired USB. A Bluetooth connection
holds one device; a wired setup can accumulate several devices through a USB hub.

On the web, a MultispeQ connects over USB/serial only. Its wireless link is
Bluetooth Classic (BR/EDR), which the browser's Web Bluetooth API cannot reach.

## Before the first field session

- Attach a [workbook](/guide/experiments/workbooks) containing a MultispeQ
  protocol to the experiment.
- Open the experiment in the app while online so the pinned workbook version is
  available offline.
- Run a test measurement and confirm that the expected questions, protocol, and
  analysis appear.

If several devices should run together, see
[Multi-device measurements](/guide/devices-protocols/multi-device).

For device calibration or hardware-specific service, use the device
manufacturer's support material; openJII controls the measurement workflow and
data handling, not the device's calibration procedure.
