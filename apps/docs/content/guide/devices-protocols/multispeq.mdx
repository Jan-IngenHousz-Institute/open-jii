A **MultispeQ** is handheld measurement hardware operated by a person. openJII
sends a selected MultispeQ protocol to the device, receives the raw response,
and can run the workbook's analysis macro before you accept the result.

Use it when your method calls for measurements at chosen plants, leaves, or time
points rather than an unattended data stream.

## Connect a device

The connection flow below is not MultispeQ-specific: any supported instrument
connects through the same **Devices** sheet, whether it is attached by USB cable
or discovered over Bluetooth, and both kinds appear in the same device list.
MultispeQ is simply the most common case.

1. Turn on the device. A MultispeQ that is not discoverable becomes so when you
   hold its button for about three seconds.
2. In the openJII mobile app, open **Devices** from the device chip.
3. Allow Bluetooth access when Android asks.
4. Select **Scan**, then **Pair** beside the intended device.
5. Check that the sheet shows the device as connected. The primary device can
   also show battery level and firmware version when those values are available.

The mobile app supports Bluetooth Classic and wired USB connections. A Bluetooth
setup holds one device; a wired setup can accumulate several devices through a
USB hub.

## Before the first field session

- Attach a [workbook](/guide/experiments/workbooks) containing a MultispeQ
  protocol to the experiment.
- Open the experiment in the app while online so the pinned workbook version is
  available offline.
- Run a test measurement and confirm that the expected questions, protocol, and
  analysis appear.

If several devices should run together, see
[Multi-device measurements](/guide/devices-protocols/multi-device).

For instrument calibration or hardware-specific service, use the instrument
manufacturer's support material; openJII controls the measurement workflow and
data handling, not the instrument's calibration procedure.
