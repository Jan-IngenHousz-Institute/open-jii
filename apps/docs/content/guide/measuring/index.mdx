The openJII mobile app is the field companion for experiment workbooks. It can
guide questions and instructions, run MultispeQ protocols, process results, save
measurements locally, and upload them when connectivity is available.

## Field workflow

1. [Install the app](/guide/measuring/installing-app) and sign in while online.
2. Open each experiment you need so its pinned workbook version is cached.
3. Connect the MultispeQ and run a test before leaving.
4. [Take measurements](/guide/measuring/taking-measurements) by following the
   workbook one step at a time.
5. Use **Recent** to see whether each reading is queued, uploading, synced, or
   failed.
6. If connectivity drops, follow [Working offline & syncing](/guide/measuring/offline-sync).

<Callout type="info">
  A workbook can also contain a questions-only flow. In that case the app collects observations
  without requiring a connected sensor.
</Callout>

## What is recorded

A saved measurement includes the experiment and protocol identifiers, device
result, answers and annotations when present, contributor identifier, and a
server-synchronized UTC timestamp with the resolved timezone. The current mobile
code uses location permission to resolve timezone; it does not add latitude and
longitude to the measurement payload.

After sync, use the web platform's [Data & Analysis](/guide/data-analysis) tools
to inspect tables, create visualizations, and export datasets.
